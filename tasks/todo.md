# 작업 계획: WoZ 화면 제거 및 익스텐션 단독화

## 요구사항 요약
- WoZ 절차가 사라졌으므로 실험자/피험자 분리 화면을 제거한다.
- 기존 evidence classifier 기반 익스텐션 흐름은 유지한다.
- 서버 세션 동기화와 원격 실험 인터페이스 관련 파일을 정리한다.

## 접근 방식
- `experiment-interface/` 정적 화면과 하위 API를 제거한다.
- WoZ 전용 서버 API와 Vercel 설정을 제거한다.
- 테스트는 익스텐션 smoke 테스트만 실행하도록 단순화한다.
- README는 익스텐션 단독 사용법만 남긴다.
- 익스텐션 설정 UI에서 더 이상 쓰지 않는 API 키 필드와 피험자 문구를 제거한다.

## 검증 전략
- `npm test`가 익스텐션 smoke 테스트만 실행하고 통과해야 한다.
- 코드/문서/테스트에서 WoZ 화면, 실험자/피험자 화면, 원격 세션 API 참조가 남지 않아야 한다.

## 검토
- `experiment-interface/` 화면 및 하위 API를 제거했다.
- WoZ 전용 루트 API와 Vercel 설정을 제거했다.
- `package.json`의 테스트를 익스텐션 smoke 테스트 단독 실행으로 단순화했다.
- `README.md`는 익스텐션 단독 구조만 설명하도록 갱신했다.
- `src/content.js`에서 사용하지 않는 API 키 저장 UI와 피험자 문구를 제거했다.
- 검증: `npm test` 통과.
- 잔여 참조 확인: 코드/문서/테스트 범위에서 WoZ, 실험자/피험자 화면, 원격 세션 API 관련 검색 결과 없음.
